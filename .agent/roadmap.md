# figure-verification — roadmap

Local "verified-plot" PoC. A weak local LLM only PROPOSES a restricted JSON chart spec (VPlot); a separate trusted verifier deterministically recomputes the plotted data from the source CSV, runs structured checks, blocks charts whose spec, encoding, policy, or dataset binding fail those checks, and renders only verified charts with a provenance certificate (dataset hash, spec hash, plotted-table hash, passed checks).

- **Scope-seed**: `.agent/outline.md` — the original outline as 16 verbatim seed steps "Milestone 0..15" (commit `9d09ecb`). The ledger below maps each routine-milestone `M<m>` to those steps; read the relevant seed step on demand when planning a milestone.
- **Stack + determinism invariants**: `.agent/memory.md` (Stack / Determinism sections) — researched SOTA, deliberately overriding the outline's human-popular defaults.
- **Data-flow (trust spine)**: the untrusted model proposes ONLY a VPlot spec (transforms + encoding + declared `dataset.hash`) — never plotted values. The verifier recomputes ALL plotted data; the renderer inlines only that. So lies needing model-supplied data (the seed's "plots a value ≠ recomputation") are impossible by construction, not checks; checks target spec/encoding/policy/dataset-binding consistency. (The seed's `aggregates_match_recomputation` example carries a model-supplied value — a seed inconsistency, resolved here.)
- **Modest claim** (hold the line): verified = {validated spec, the independently recomputed plotted table, the emitted Vega-Lite inlining only that table, the provenance badge} are mutually consistent and the checks passed. Trusted, NOT verified (TCB): `vl-convert`/Vega, SVG rasterization, browser, pixels — trusted to render verified data faithfully, not proven to.
- **Quality gate** (M1.1 wires it; every WORK-UNIT VERIFY runs it, all green, touched scripts exit clean): `ruff format --check .` · `ruff check .` · `mypy` · `pytest` — all via `uv run --locked` (the lockfile, not a newer floor-satisfying release, pins the gate).

### M1.2a — VPlot v0.1 schema + fail-closed decoder
Research is done — read `memory.md` Stack "msgspec confirmed behaviors" (cited below by number) and transcribe; do not re-derive. pyproject: add `dependencies = ["msgspec>=0.21,<1"]`, dev `jsonschema>=4.21`, and a `[[tool.mypy.overrides]]` `ignore_missing_imports` for `jsonschema.*` (test-only, no `py.typed`); re-lock.
- `verifier/schema.py` — frozen, `kw_only`, fail-closed `msgspec.Struct`s. A `_Base(Struct, frozen=True, forbid_unknown_fields=True, kw_only=True)` carries shared config (`forbid_unknown_fields` propagates at runtime); every concrete struct repeats `frozen=True, kw_only=True` (finding 1 — frozen for mypy, kw_only does NOT inherit). `Meta`-constrained aliases (each leads with `(?!.*[\r\n])` — `$` alone admits a trailing newline, confirmed at runtime): `FieldName` = `^(?!.*[\r\n])[A-Za-z_][A-Za-z0-9_]*$` ≤64; `DatasetName` = `^(?!.*[\r\n])[A-Za-z0-9][A-Za-z0-9._-]*\.csv$` ≤128; `DatasetHash` = `^(?!.*[\r\n])sha256:[0-9a-f]{64}$`. `Literal`s: `Mark`{bar,line,scatter}, `ChannelType`{quantitative,temporal,ordinal,nominal}, `AggFn`{sum,mean,count,min,max}, `CmpOp`{eq,ne,lt,le,gt,ge}, `SortOrder`{ascending,descending}. `FilterValue = int | Annotated[str, Meta(max_length=128)]` — NOT Decimal (finding 3: rejects float/bool/null tokens at decode, keeping the M1.4 re-encode exact; decimals travel as bounded strings). Structs: `Channel{field, kind=field(name="type")}` (finding 6), `Encoding{x, y, color=None}`, `Dataset{name, hash}`; transforms tagged on `op` — `Select{fields: tuple[FieldName, ...] 1..64}`, `Filter{field, cmp, value:FilterValue}`, `GroupBy{keys: tuple[FieldName, ...] 1..32}`, `Aggregate{measures: tuple[Measure, ...] 1..32}` w/ `Measure{field, fn, output=field(name="as")}`, `Sort{by: tuple[SortKey, ...] 1..32}` w/ `SortKey{field, order}`; `Transform = Select|Filter|GroupBy|Aggregate|Sort`; `VPlotSpec{version:Literal["vplot-0.1"], dataset, transform: tuple[Transform, ...] ≤64, mark, encoding}`. Arrays are bounded tuples, not lists (finding 7 — deep immutability + hashable specs). One module-level `Decoder(VPlotSpec)`. `decode_spec(raw)` = decode, then a stdlib `json.loads(object_pairs_hook=…)` dup-key pre-scan raising `ValidationError` (finding 4). `json_schema()` = `{**msgspec.json.schema(VPlotSpec), "$schema": <draft-2020-12 URI>}` — `$schema` stamped LAST so a future msgspec emission cannot override it (finding 5: today msgspec emits none); `json_schema_text()` = `json.dumps(…, indent=2, ensure_ascii=False)+"\n"`.
- `schema/vplot-0.1.schema.json` — committed golden = `json_schema_text()` output.
- `tests/test_schema.py` (replaces `test_smoke.py`) — deterministic only (fuzz → M1.2b): positive decode of a full spec; parametrized rejection per error layer (unknown field/mark/op/agg-fn/channel-type/cmp; a float/bool/null `value` — but a numeric STRING `value` like `"1.2"` DECODES OK, its numeric validity being an M1.4 semantic check; wrong `version`; pattern/length/empty-list violation; a trailing newline or control char in a field/dataset/hash; malformed JSON → `DecodeError`; non-object input); duplicate key top-level and nested; immutability (attribute set raises `AttributeError`, and `hash(spec)` succeeds — finding 7); positional construction of each struct raises `TypeError` (kw_only, finding 1); a `__struct_config__` introspection asserting every struct is frozen + `forbid_unknown_fields` (finding 1); the golden schema is Draft-2020-12 valid (`jsonschema.Draft202012Validator.check_schema`) and byte-equals the committed file.
- **Accept**: valid specs → typed objects, never partial/coerced; every listed rejection raises at decode; dup keys rejected; structs frozen + fail-closed by introspection; golden schema Draft-2020-12-valid + byte-stable; quality gate green at 100% branch coverage.

#### M1.4a — Canonical forms + provenance hashing (`verifier/canon.py`)
Evaluator-independent foundation (no transform logic; M1.4b/c/d import it).
- Value model: `Cell = Decimal | str | None` (numeric / temporal+string canonical text / the one null token); `Column`(name, `kind`∈numeric|temporal|string, `scale` for numeric, `granularity` for temporal — NO unit/label: display metadata stays in the manifest so it never enters the data hash); `Table`(columns, rows already in canonical total order — M1.4c's closure produces them).
- `serialize_table` → typed-NDJSON: a header JSON array of self-describing `name:numeric:<scale>` / `name:temporal:<granularity>` / `name:string` descriptors, then one compact JSON row-array each (`ensure_ascii=False`, `separators=(",",":")`), numerics quantized to exactly `scale` places HALF_EVEN, null → JSON `null` (unambiguous vs the string `"null"`), trailing `\n`.
- Hashes (SHA-256 over text, never Arrow/Parquet): `hash_dataset(csv_bytes)` = raw bytes, `sha256:`-prefixed, NO domain tag (≡ the spec's declared `dataset.hash` source identity; sensitive to row-order/CRLF/BOM by design); `hash_table`/`hash_spec`/`hash_manifest` via `_digest(domain, payload)` = sha256 of a `vplot-<domain>/<CANON_VERSION>\n` tag + payload (domain a closed slash-free `Literal` → the tag/payload split is injective; a serialization-format bump invalidates stale tagged hashes; the raw dataset hash stays format-free). `hash_spec` = digest of `Encoder(order="deterministic").encode(spec)` (finding 8: definition-order fields, Decimal→string, no Unicode pass) — byte-faithful, NO NFC (the eval+oracle compare filter strings verbatim by code-point order, §3/§4/§10; folding here would collide specs selecting different rows). `hash_manifest` = digest of the raw manifest bytes.
- `CANON_VERSION` tag; golden hex vectors (tests) lock the canonical bytes against drift from any source (msgspec/Decimal/Python/locale) — no runtime UCD assertion. `runtime_versions() → Versions`(canon_version, python, msgspec, unidata) — the M1.6 badge stamps these (unidata = informational provenance).
- `tests/test_canon.py`: dataset hash flips on any source byte change (row-order/CRLF/BOM), stable on identity; table hash flips on cell/column/scale change, equal for tables that serialize identically; spec hash flips on a spec edit, DISTINGUISHES NFC variants (compose vs decompose a combining mark → different hashes), stable across re-encode; manifest hash flips on byte change; serialize emits raw UTF-8 + escapes JSON control/meta, table cells byte-faithful, a too-long row rejected; golden hex vectors pin all four digests; `runtime_versions` populated; all four hashes byte-identical across two `PYTHONHASHSEED` subprocesses (memory Ops: seed varies only via subprocess).
- **Accept**: dataset hash = tag-free raw-byte identity (order-sensitive); table/spec/manifest domain-tagged; spec hash byte-faithful (distinguishes NFC variants, no UCD assertion); golden hex vectors lock all four canonical forms; every hash deterministic across PYTHONHASHSEED subprocesses; gate green at 100% branch coverage.

#### M1.4c — Deterministic evaluator: transform pipeline (`verifier/eval.py`)
Imports `canon` (Cell/Column/Table) + `ingest` (`load_table`, `Manifest`) + `errors` (`VerificationError`, its canonical home). Hand-rolled, Decimal-exact (memory invariants: no float, no Kahan), implementing `VPlot_SEMANTICS.md` §3–6. Applies a validated spec's transforms to the ingested source table, closing with the total sort. Reads: `canon.py` + the just-built `ingest.py`, `schema.py` (transform types — `Select`/`Filter`/`GroupBy`/`Aggregate`/`Measure`/`Sort`/`SortKey`/`CmpOp`/`AggFn`), `examples/index.json` (the good + eval-bad corpus), `tests/test_canon.py` (test style), §3–6. (b/c were one `eval` unit that overflowed a window in DESIGN alone → this blueprint is pre-derived; implement it. If context nears ~75% before the gate is green, STOP and report for a test-split re-plan rather than commit under 100% coverage.)
- `evaluate(spec, manifest, csv_bytes) → canon.Table` — sole public entry: `ingest.load_table` → apply `spec.transform` ops in order → total-sort closure. `Table.columns` = the plotted names+types (no separate `plan_columns`); M1.5 resolves unit/label via the spec→measure→source-field lineage + manifest (§7), NOT off `Table.columns` (no unit/label there → display metadata out of the data hash). Each op first binds against the current schema: referenced field exists (`schema.fields_exist`); a `filter` value coerces to the field kind (`schema.field_types_match` / `filter.value_type`). Semantic failures raise `VerificationError(msg, *, check)`.
- Pipeline (match on `op`, §4):
  - `select` = projection, no dedup; fields exist + distinct (`select.fields_distinct`).
  - `filter` = 3-valued: `_coerce_filter_value(value, column)` per §3's table — (int,numeric)→`Decimal(int)`@scale; (string,numeric)→`Decimal(string)` exact-at-scale (over-precise→error); (string,temporal)→canonical-ISO parse (bad format→error); (string,string)→verbatim; (int,temporal) and (int,string)→error; ALL failures raise `filter.value_type`. The string→numeric/string→temporal parses reuse ingest's `_coerce_numeric`/`_coerce_temporal` precision/canonicality (re-tagged `filter.value_type`); §3 bounds filter literals by parse+precision+format, NOT the table magnitude bound; compare cell vs value with `_CMP[cmp]` (operator module: eq/ne/lt/le/gt/ge); a null or incomparable cell drops the row.
  - `group_by` = null key its own group; keys exist + distinct (`group_by.keys_distinct`); valid only immediately before an `aggregate` (else `transform.group_by_placement`).
  - `aggregate` per group: `count` = non-null count (integer, scale 0; zero non-nulls → 0, NOT null) / `sum`,`mean` (numeric input only; `mean` via `mean_at_scale` below) / `min`,`max` (inherit the source scale); `sum`/`min`/`max` over zero non-nulls → null (count → 0). No preceding `group_by` → one whole-table row. Each measure → an output column named `output`, scale = source scale (count → 0); `output` names unique + not colliding with a group key (`aggregate.output_unique`).
  - `sort` = keys DISTINCT (`sort.fields_distinct`); no reorder/schema change here — the §6 closure applies the active sort (a superseded or pre-aggregate sort is otherwise a no-op).
  - helpers (Decimal-exact, no float — verified scratchpad): `mean_at_scale(total: Decimal, count: int, scale: int) → Decimal` = `Fraction(total)/count`, `round(_, scale)` (a Fraction, ROUND_HALF_EVEN — ONE rounding, no Decimal-division double-round); then `scaled = round(_, scale) * 10**scale` (a Fraction; `assert scaled.denominator == 1` — the rounded value is an exact multiple of `10**-scale`) → `_scaled_int_to_decimal(scaled.numerator, scale)`; `_scaled_int_to_decimal(scaled: int, scale: int) → Decimal((sign, digits-of-abs, -scale))`; `sum` = plain Decimal `sum` over the same-scale cells (exact). Expose `mean_at_scale` for the M1.4d oracle to reuse (parity).
  - distinctness §5 = the select / sort / group_by / aggregate-output checks above.
- Total-sort closure §6: active sort = the LAST `sort` op with no later `aggregate` (an intervening aggregate resets it — track while walking the ops); active-sort fields must exist in the plotted schema (`sort.field_in_plotted_table`); sort by the active keys (each in its declared `order`) then every remaining column ascending, null GREATEST → a total order → permutation-invariant hash. Implement as a multi-pass STABLE sort, least-significant key first; per-column key `(cell is None, sentinel-if-null-else-cell)` with the null sentinel typed by column kind (`Decimal(0)` numeric / `""` temporal+string) so null-vs-null never compares `None<None` regardless of CPython sort internals (bare `(cell is None, cell)` also works empirically — verified — but the typed sentinel is the soundness-core choice); `reverse=True` for a descending key.
