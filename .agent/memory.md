## Stack (M1 trusted core) — researched SOTA, deliberately overriding the outline's human-popular defaults
- Runtime: `uv` src-layout single pkg `verifier` (+`py.typed`, `uv_build`), committed `uv.lock`. `requires-python = ">=3.13,<3.14"` AND committed `.python-version` = 3.13 both pin the 3.13.x line. Cap (not just a floor) for reproducibility: msgspec ships per-interpreter compiled wheels and stdlib behavior can drift across minors, so a bare `>=3.13` floor would silently admit an untested interpreter. The pin is necessary, NOT sufficient — determinism is ASSERTED by golden hex vectors (the actual canonical-form guard, see Hashing), and the runtime stamps full Python/msgspec/unidata versions + `canon_version` into the VCert for provenance (M1.4/M1.6). Both layers (host `.venv-host`, container `.venv`) honor `.python-version`. FastAPI/service deferred to M2.
- Schema: **msgspec** (NOT Pydantic) — strict/fail-closed, `forbid_unknown_fields` on every `Struct`, transform ops = tagged union (`tag_field="op"`), `Literal` enums, module-level `Decoder`, `json.schema` export golden-snapshotted.
  - **msgspec confirmed behaviors** (empirically pinned for our msgspec; this cost a full window of research — read and transcribe, don't re-derive): (1) `frozen`/`forbid_unknown_fields` propagate to subclasses at RUNTIME but `kw_only` does NOT; and mypy's dataclass_transform reads each class's own kwargs — so EVERY concrete struct must repeat `frozen=True` (for `mypy --strict`, else "non-frozen dataclass cannot inherit from frozen") AND `kw_only=True` (runtime kw-only, since it does not inherit). A `__struct_config__` test backstops frozen+forbid_unknown_fields but does NOT expose `kw_only`, so a separate test asserts positional construction raises `TypeError`. (2) For the transform tagged union, set `tag_field="op"` + an explicit lowercase `tag` per member (else msgspec defaults the tag to the class name and the field name to `type`). (3) Strict mode rejects float→int and bool→int but ACCEPTS a JSON float→`Decimal` — so model spec numerics as `int | str` (NOT Decimal) to reject float/bool tokens at decode; decimals travel as strings, lifted per manifest at eval; the int is further bounded to signed int64 (`Meta(ge=-(2**63), le=2**63-1)`) — out-of-range or fractional numbers travel as the bounded string. (4) Duplicate object keys silently LAST-WINS, no switch — reject via a stdlib `json.loads(raw, object_pairs_hook=…)` pre-scan after the msgspec decode (input is already well-formed JSON). (5) `msgspec.json.schema()` emits NO `$schema` (stamp the Draft-2020-12 URI ourselves), is deterministic cross-process, sorts `Literal` enums alphabetically, renders a tagged union as `anyOf`+OpenAPI `discriminator`+single-element `enum`, maps `forbid_unknown_fields`→`additionalProperties:false`. The exported golden is ADVISORY, not the gate — JSON Schema `integer` admits `1.0`/`1e3` (zero-fraction floats) that strict decode rejects, so `decode_spec` stays authoritative; pop+re-add `$schema` so it sorts last even if a future msgspec emits one. (6) `msgspec.field(name=…)` renames in decode+encode+schema and rejects the un-renamed key (used for `type`→`kind`, `as`→`output`); when an attribute is itself named `field` (`Channel.field`, `Measure.field`, …) call it as `msgspec.field(…)` via the module — the bare `field:` annotation shadows the imported `field` for mypy ("str not callable"), though it still works at runtime (annotation-only names don't bind). (7) A frozen struct holding a `list` is only SHALLOWLY immutable (`spec.transform.append(…)` still mutates) and unhashable — so model every JSON array as a bounded `tuple[T, ...]` (`Annotated[..., Meta(min_length/max_length)]`; msgspec decodes JSON arrays to tuples and enforces the length), making specs DEEPLY immutable and fully hashable. Test immutability via an attribute set raising `AttributeError`. (8) `Encoder(order="deterministic")` keeps struct field order while sorting dict/set keys, renders Decimal→string, does NO Unicode normalization (→ the M1.4 spec hash).
- Evaluator: **hand-rolled pure-Python** (NOT pandas/polars) — the trusted core; total control of row/col order, nulls, float accumulation.
- Hashing — all SHA-256 over **text** canonical forms, NEVER Parquet/Arrow bytes (version-coupled): table = typed-NDJSON (header `name:type`, fixed col order, total-sort rows, fixed per-type formatters, one null sentinel); spec = msgspec re-encode of the validated struct (definition-order fields, pinned msgspec, byte-faithful strings — NO Unicode normalization, floats forbidden in specs, `canon_version` tag — leaner than JCS/rfc8785, sufficient since we own the encoder); dataset = raw CSV bytes (byte-exact SOURCE identity, NOT permutation-invariant); + per-column manifest hash. Only the plotted-table hash is permutation-invariant (total-sort closure).
- Oracle (dev/test only): **DuckDB** `threads=1`, columns as matching `DECIMAL`, must reproduce byte-identical canonical tables on goldens; ops it cannot match bit-for-bit → logged tolerance cross-check (dual-engine determinism check).
- Renderer: **vl-convert-python** (pinned `vl_version`) → static **SVG** as the canonical/hashed artifact; hand-built Vega-Lite (NOT Altair) emitted via a POSITIVE allowlist schema (drop every key outside the generated subset — excludes top-level `url`/`datasets`/`transform`/`params`/`expr`/`href`/`tooltip`/image-`url`/`loader`/`signals` AND encoding-level `aggregate`/`bin`/`stack`/`impute`/`sort`/`scale.domain` by construction, with implicit Vega-Lite stack/sort nulled — see Determinism invariants), inline ONLY evaluator `data.values`; VCert v0.1 badge composed in trusted Python; optional non-canonical interactive HTML (`bundle=True, actions=False`, offline), off the hash chain. Vendor + register a font for SVG reproducibility. Output-scan test: no `<script>`/external `href`/`src`/`url`, zero fetches.
- Lint/format: **ruff** (incl. S/T20/DTZ). Types: **mypy --strict** (hermetic, pure-Python; NOT `ty` — its silent unimplemented-check gaps are unsafe for a soundness core; `ty`/pyright editor-only). Tests: **pytest** + **Hypothesis** (property: 3-hash permutation-invariance + cross-run stability, aggregate-vs-recompute; CI `derandomize=True`, `deadline=None`) + **syrupy** goldens (assert-only in CI).

## Determinism invariants (M1 implementation MUST hold; migrate into tests at M1, prune at M1 review)
- Data-flow invariant: the model proposes ONLY the spec (transforms + encoding + declared `dataset.hash`); the verifier computes ALL plotted data and the renderer inlines only that — no model-supplied value ever reaches a chart. `dataset.hash_matches_source` binds the spec to the exact bytes (source path confined to `data/`). Scope limit (claim honesty): checks prove mechanical consistency (spec/encoding/policy/binding), NOT representativeness or intent — a valid cherry-picked `filter` passes; the VCert badge discloses every applied filter/sort (M1.6) so a reader sees the selected subset. Axis units/labels are sourced from the trusted manifest, never the spec — present and correct by construction.
- Parse CSV cells as text → coerce by the M1.3 manifest (the trusted schema; CSV alone has no types/units/labels).
- Numeric model is singular: `Decimal`/scaled-int with a declared per-measure scale + `ROUND_HALF_EVEN`; aggregation/division are exact-then-quantize (associative → hash-stable), so NO float math and NO Kahan. The DuckDB oracle must match this `DECIMAL` precision/rounding for hash equality.
- Close every transform with an explicit TOTAL sort over all columns (declared `sort` first, fixed tiebreak); NaN never a sort key; one canonical null token.
- Validate spec against schema BEFORE recompute, so only allowlisted ops reach the evaluator (upholds `security.no_arbitrary_code`). No `eval`/`exec`/dynamic import anywhere.
- Renderer pins `vl_version` + a bundled font for byte-stable SVG; one spec per render (Vega element-id counter is process-global). It emits NO Vega-Lite data transform: drop encoding-level `aggregate`/`bin`/`stack`/`impute`/`sort`, any `scale.domain`/`scale.type` override, and top-level `transform`; explicitly set `stack:null` + an explicit `sort` matching the recomputed row order to defeat Vega-Lite's IMPLICIT stacking/sorting — else displayed marks diverge from the inlined recomputed table and the inline-equals-pixels claim breaks.

## M1.3 corpus + manifest (golden inputs)
- Trusted manifest = `data/schemas/<stem>.json` (`columns` ordered = CSV header; numeric→`scale`≥0, temporal→`granularity`∈date|datetime, optional `unit`/`label`). `aqi` left unit-less ON PURPOSE = the B13 missing-unit fixture.
- `month` (`YYYY-MM`) = `string`/`ordinal`, NOT `temporal` (semantics temporal = `YYYY-MM-DD`/datetime only, §2) → M1.4/M1.5/M1.6 treat month as ordinal; `weather.date` is the temporal exemplar.
- `NA` (region/city) = literal string; only an empty cell is null (§2) → the hand-rolled M1.4 loader must NOT adopt pandas na_values defaults.
- `examples/index.json` = corpus source of truth (good=10, 1/intent + semantically valid; bad=18 with `decodes`/`layer`/`check`/`reason`/`caught_by`; `enforced_by_construction` = checks with no model-spec attack vector). Good specs bake the live `sha256(raw CSV)` → editing a CSV breaks that binding BY DESIGN.
- Bad-spec mono-causality is COUNTERFACTUAL: each fails exactly one way even if you imagine fixing that defect → choose dataset/fields so no second defect lurks (float-token fixture on a scale-1 column so the value is valid-as-string; a missing field referenced at ONE site only, e.g. a filter, not also in `select`+encoding where two checks fire). `caught_by` is PROVISIONAL — §9 keeps the semantic layer JOINT (M1.4 eval + M1.5 checks), so don't over-pin one milestone where eval would catch it first. M1.3 tests cover decode + static props (live hashes, scale=actual-decimals) only; single-defect-ness + the catching layer get asserted by the M1.4/M1.5 suites when built.
- Good specs aggregate only unit-bearing fields with unit-PRESERVING fns (`sum`/`mean`/`min`/`max`); a derived column (aggregate `as`) inherits `unit`/`label` through its measure to the source (VPlot_SEMANTICS §7). `count` is dimensionless → would trip `label.quantitative_units_present`; no golden uses it.

## M1.4a canon (built) — contracts the rest of M1 consumes
- 4 `sha256:<hex>` hashes over TEXT (`verifier/canon.py`): `hash_dataset` = raw CSV bytes, NO tag (≡ the spec's declared `dataset.hash`; order/CRLF/BOM-sensitive); `hash_table`/`hash_spec`/`hash_manifest` domain-tagged `vplot-<domain>/<CANON_VERSION>\n`+payload (`CANON_VERSION="canon-0.1"`; a bump kills stale tagged hashes, the dataset hash stays format-free).
- ALL four hashes are byte-FAITHFUL — NO Unicode normalization anywhere (codex-review reversed an earlier NFC-on-`hash_spec` design): the evaluator + DuckDB oracle compare filter strings verbatim by UTF-8 code-point order (VPlot_SEMANTICS §3/§4/§10), so NFC-folding the spec hash collided specs selecting DIFFERENT rows and would have forced NFC into eval+oracle too (breaking oracle parity + the byte-faithful-table invariant). String-canonicalization stays DEFERRED to M1.4c/M1.6. `hash_spec` = a deterministic msgspec re-encode of the validated struct (folds JSON surface noise — whitespace/key-order/escaping — to one form, no Unicode pass). No runtime UCD assertion (retired with NFC); `runtime_versions()→Versions`(canon_version, python, msgspec, unidata) still stamps the M1.6 badge, unidata as informational provenance only.
- Golden hex vectors (tests; fixed input→fixed digest for dataset/table/spec/manifest) LOCK the canonical bytes against drift from ANY source (msgspec, Decimal, Python, locale) — stronger than a runtime msgspec-version pin (already lockfile-pinned); under a fixed CANON_VERSION any encoder/runtime drift fails loudly. Re-derive with the scratchpad `compute_goldens.py` pattern (redirect stdout to a file — Headroom compresses inline tool output).
- `Column` = kind-tagged union (`NumericColumn{name,scale}|TemporalColumn{name,granularity}|StringColumn{name}`, `.kind` ClassVar) carrying NO unit/label → display metadata stays in the manifest, out of the table hash (M1.5 resolves unit/label via spec→measure→manifest, not off `Table.columns`). canon does NOT sort: permutation-invariance is M1.4c's total-sort-closure duty (`serialize_table` preserves the given row order, fails loud on width/type mismatch). `Cell = Decimal|str|None`; numerics render as fixed-point JSON number tokens at exactly `scale` places HALF_EVEN, -0 folded.
- PEP695 `type` aliases (incl. an untagged struct union) work as msgspec Struct FIELD types when the struct is never decoded (canon `Cell`/`Column` inside `Table`).
