## M5 planning decisions (formal + provenance hardening)
- Operator policy (M5.1e) supersedes M4's bare-int-guard decision: frozen stdlib `Settings`
  owns every service/core/proposer/cache/admission scalar; exact signed-64-bit resource integers,
  eager derived `settings.limits`, and overflow-safe per-item cache compatibility validate at
  startup. `from_env` is the sole ambient read; core/service work consumes that one snapshot.
- Formal scope: Z3 checks bounded CONCRETE table/builder obligations (canonical order, bar zero, discrete legend exactness). It is a second trusted checker, not a universal proof; `unknown`/timeout/error blocks. Every public `verified=true`, including verify-only, passes SMT; render reuses that exact prepared Vega without rebuild/re-solve. Each worker invocation owns an explicit Z3 Context because contexts are not thread-safe. Aggregation stays deterministic recomputation. Check method travels in verdict + VCert; Z3 version joins TCB.
- Resource order: bounded reads/rows/cells/output/model replies + payload-byte LRUs + global rate/active-job admission gate the corresponding model/Z3/native work; an active-job permit spans proposer wait, CPU work, and archive commit. A cancelled request retains it until any uncancellable worker exits. Transactional LOGICAL archive quota gates commit. Policy failure = structured non-chart outcome; cache/archive bounds make no Python-object/SQLite-page/filesystem-overhead claim, and archive quota refuses without eviction.
- Signing: VCert v0.2 binds dataset/manifest/spec/table + exact emitted Vega hashes (SVG remains display TCB), then DSSE PAE authenticates exact payload bytes + application type; Ed25519 via PyCA. `keyid` only selects candidate keys; current signer + explicitly configured historical pins form trust, never archive/public-key presence. Claim = possession of independently pinned public key, with no PKI/operator-identity/transparency/timestamp-authority implication. Verifier parses the SAME payload bytes it verified (DSSE load-bearing rule).
- Archive: stdlib SQLite STRICT, rollback-journal DELETE + FULL sync, defensive/trusted-schema controls, one transaction/bundle, bounded streaming blob reads, content digests rechecked. Explicitly avoid WAL: bundled SQLite predates the 2026 WAL-reset fix and one-writer provenance needs atomicity more than read/write concurrency. VCert + each classified outcome-bearing admitted attempt manifest are DSSE-signed; pre-admission refusals, pre-outcome disconnects, crashes, unclassified faults, and archive faults are excluded. Signatures detect modification under an externally pinned key, NOT deletion/completeness. Raw source/prompt/sample bytes remain operator-local, absent from unauthenticated HTTP.
- Replay: exact raw CSV+manifest snapshots are authority; mutable `data_dir`, stored plotted values, and model reruns are excluded. Verify blob hashes + DSSE first, then recompute with current code. `/replay/{plot_id}` selects the lowest signed successful attempt ID that binds that plot; archive completeness remains unclaimed. Same-version equality is required for `exact`; resource/solver failure + tool/key drift are reported. Exact replay may regenerate SVG + repopulate ephemeral chart cache; SVG/pixels remain display TCB diagnostics.
- Renderer probe: explicit nominal color domain `[]` for all-null data and numeric ordinal domains both compile under pinned Vega-Lite 5.21; M5.2d may transcribe these forms without re-probing.

## M1 lessons (forward; unit blow-by-blow lives in git — `git log --grep "(M1[. ]"`)
Process / right-sizing (rule itself → roadmap; it binds M2+ planning):
- Overflow modes seen, with remedies: design-too-wide → split at a coverage-safe seam; re-derivation despite a complete recipe → TRANSCRIBE, don't re-derive (trust verified goldens); probing a native dep in the implementing window → isolate probes to scratch sessions and pre-bake a gate-validated recipe; TOO-MANY-modules/layers in one unit (M4.1 chart surface: 6 src + 6 test files spanning storage+production+serving, design derived from scratch, overflowed mid-tests) → the ~one-module rule binds cross-LAYER; split by layer (render-leaf / store-LRU / capture+serve), front-load the subtle module (the LRU) alone in its own window, and pre-bake each sub-unit's transcription recipe in the roadmap so the re-plan cost is paid once. A separate additive method (put_chart, leaving put() intact) decouples a store extension from its pipeline caller, enabling that isolation cleanly.
- An investigation-heavy codex-review can overflow while the fix stays small → capture the verified fix as a re-appliable patch/recipe + verdict, splitting investigation from application.
- A claim-narrowing fix must SWEEP every restatement of the phrase across src+tests+memory — one narrowed line beside stale twins reads fixed but isn't.
Test design:
- A green INTERNAL gate (lint/type/100%-branch/structural asserts) does NOT prove EXTERNAL-contract validity → test output artifacts against the real consumer (jsonschema; a full vl-convert COMPILE — now a standing gate over all good specs).
- 100% BRANCH coverage misses data-DOMAIN gaps (the prec-28 rounding bug) and cannot pin LOOKUP-TABLE entries → regression-test the real data path at hostile magnitudes; matrix-test an enumerable key-space against the spec restated in-test (non-circular oracle).
- 100% coverage EXECUTES a line without pinning its EFFECT: an LRU recency refresh (`move_to_end`) runs on every put/get yet stays UNPINNED unless a test re-puts an ALREADY-PRESENT key or asserts a READ reorders eviction — re-putting an already-EVICTED key is a fresh insert exercising no refresh (the M4.1b vacuity codex caught). Pin recency with an `html_cap>=2` witness, and PROVE non-vacuous by deleting the refresh + confirming the test FAILS (mutation check), then reverting.
- Decimal aggregation must be context-free EXACT (`Fraction`/scaled-int), never bare `sum()`/`+` — the ambient context (prec 28) rounds intermediates → accumulation-order-dependent, breaking exactness + permutation-invariance.
- Order-sensitivity as a Hypothesis property is HOLLOW (tautological) → one concrete anchor witness; a PYTHONHASHSEED-stability guard needs TIE-HEAVY rows + several seeds else it passes vacuously; "proves" overstates a bounded sample → "samples / finds no counterexample".
- Direct-helper test fixtures must be REALIZABLE pipeline states (not phantom ones); an acceptance criterion naming a multi-step scenario needs an END-TO-END pin through the public entry, not only the algorithm helper.
- cast-not-assert: narrow `X | None` via `cast` at a gated use site — an `assert`'s never-taken branch fails the 100% gate. msgspec `Meta` constraints enforce at DECODE only (`structs.replace`/direct construction skip them) → any security gate resting on a decoded-shape invariant must re-hold at the use site (path confinement = authoritative `resolve()`+`is_relative_to`). BUT when a model string enters a path only via `Path(name).stem` or a `fullmatch` id regex, confinement is BY CONSTRUCTION → NO runtime `is_relative_to` branch (unreachable → fails warn_unreachable + 100%-branch); the authoritative form is needed only where the WHOLE model string builds the path (checks.py CSV). M2.2 dropped its roadmap-planned confinement branch on this basis (pipeline.py manifest path = `.stem`).
- Per-element uniqueness ≠ global acyclicity: a backward walk over a schema-REBUILDING pipeline keys on (name, POSITION), never name alone — else non-terminating or unsound on reused output names.
- A green parity suite ≠ a correct oracle: a probe-validated draft re-parsed source through the banned forced-DECIMAL CAST and clean goldens hid it → verify the oracle INGESTS the shared coercer's cells, and feed adversarial synthetics beyond the golden shapes.
- An equivalence differential must compare FULL messages + the cross-product of independent failure modes — a check-tag-only diff missed a magnitude/precision precedence flip.
- A test embedding a symbol name as a substring → do renames via TARGETED edits, never `replace_all`.
Refuted probes (do NOT re-investigate; evidence in git): DuckDB `executemany` does NOT route large in-domain Decimals through float64; whole-table + all-null-group aggregates agree with eval; native Decimal binds correctly across its full range EXCEPT positive-exponent literals (the oracle filter path coerces via `ingest._coerce_numeric`, exp≤0). Planning-era "one spec per render (Vega element-id counter)" concern refuted — in-process SVG byte-determinism holds (tested).
Deferred by design: evaluator DoS budgets (work is O(cols·N log N) over the TRUSTED bounded source; the render stage likewise builds full-table Vega-Lite JSON + SVG/HTML over that same trusted bounded source — the model picks only the dataset NAME, never supplies CSV bytes — so a render/output-size budget (rows/cells/bytes) joins eval as M5 hardening, a verifier-core policy check returning a 200 failed verdict before render, not transport; untrusted-CSV/spec rate budgets likewise M5).
